# Architecture Documents

`docs/architecture/` has one stable architecture overview and one mutable migration ledger.
The short content index below is intentionally not another source of progress
truth.

## Content Index

- `docs/architecture/rem6-architecture.md` describes the simulator
  architecture: gem5 pain points, rem6 runtime shape, invariants, workspace
  responsibilities, and the evidence policy for production components.
- `docs/architecture/gem5-to-rem6-migration.md` tracks gem5-to-rem6 migration
  progress: component scores, score calculations, markdown checklists,
  migrated and missing behavior, test-anchor crosswalks, and next evidence.

Keep architecture rationale in the overview. Keep percentages, checkboxes,
proof notes, and changing migration state in the migration ledger.
